# Gold Event Model Skeletons

This scaffold prepares the repository for supervised temporal-event modelling
after CVAT annotations are exported and imported.

The existing weak-label pipeline remains unchanged. The gold workflow adds a
separate namespace:

- `temporal_module/data_gold/`
- `temporal_module/runs_gold/`

## Required Gold Input

The first required file is the canonical annotation export:

```text
temporal_module/data/gold_event_project/annotations/gold_event_intervals.csv
```

Expected columns:

```text
clip_id,event_id,event_type,start_frame,end_frame,start_seconds,end_seconds,cvat_task_name,annotator,confidence,uncertain,notes
```

Supported event classes:

- `background`
- `carry`
- `pass`
- `turnover`
- `shot`

`background` is generated by the window builder. It is not annotated in CVAT.

## Gold Temporal Feature Windows

Build model-ready numeric feature windows from existing `temporal_frames.csv`
files and imported CVAT intervals:

```cmd
python temporal_module\scripts\build_gold_event_windows.py ^
  --gold-events temporal_module\data\gold_event_project\annotations\gold_event_intervals.csv ^
  --derived-root temporal_module\data\derived ^
  --output-dir temporal_module\data_gold\event_windows ^
  --window-size 64 ^
  --stride 8 ^
  --label-region-frames 16
```

Outputs:

- `temporal_module/data_gold/event_windows/gold_event_windows.npz`
- `temporal_module/data_gold/event_windows/gold_event_windows_metadata.csv`
- `temporal_module/data_gold/event_windows/gold_event_label_distribution.csv`
- `temporal_module/data_gold/event_windows/gold_event_windows_summary.md`
- `temporal_module/data_gold/event_windows/gold_event_windows_summary.json`

If the CVAT import has not happened yet, the builder writes a clear `NOT READY`
summary instead of crashing.

## Gold BiGRU

Train the numeric-feature BiGRU:

```cmd
python temporal_module\scripts\train_gold_event_bigru.py ^
  --data temporal_module\data_gold\event_windows\gold_event_windows.npz ^
  --output-dir temporal_module\runs_gold\event_bigru ^
  --epochs 40 ^
  --seed 42
```

Outputs:

- `best_model.pt`
- `metrics.csv`
- `metrics.json`
- `confusion_matrix.csv`
- `predictions_val.csv`
- `training_summary.md`

## Gold CNN-LSTM Prototype

The visual model scaffold expects a future prepared frame-window tensor file:

```text
temporal_module/data_gold/frame_windows/gold_frame_windows.npz
```

Expected NPZ arrays:

- `X`: `[windows, frames, height, width, channels]` or `[windows, frames, channels, height, width]`
- `y`: integer labels
- `label_names`: class names

Training command:

```cmd
python temporal_module\scripts\train_gold_event_cnn_lstm.py ^
  --data temporal_module\data_gold\frame_windows\gold_frame_windows.npz ^
  --output-dir temporal_module\runs_gold\event_cnn_lstm ^
  --epochs 20 ^
  --seed 42
```

This script also writes a `NOT READY` summary when the frame-window dataset is
not present.

## Report-Safe Wording

The project preserves the original automatic pipeline and weak-label prototype,
then adds a gold-supervised modelling path. Once CVAT annotation exports are
available, manually verified event intervals can replace heuristic weak labels
for training temporal models. The BiGRU uses numeric temporal features derived
from tracks, ball motion, possession and team-shape features, while the
CNN-LSTM scaffold is reserved for future visual-frame experiments.

Until the CVAT exports are imported, these gold-model scripts should be
reported as implemented training scaffolds rather than completed supervised
experiments.
